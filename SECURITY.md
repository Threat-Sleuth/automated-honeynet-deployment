# Security policy

This repository intentionally deploys vulnerable services and weak laboratory credentials. It must be used only in isolated, authorized environments.

Do not expose the stack directly to the public Internet. Restrict access at the hypervisor, host firewall and network perimeter. Treat generated logs and datasets as potentially sensitive research data.

To report an unintended vulnerability in the deployment framework, installation scripts or supporting tooling, contact the project maintainers privately before public disclosure. Do not report the deliberately vulnerable behavior of honeypot services as a framework vulnerability.
